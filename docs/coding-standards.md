# Coding Standards

- Use TypeScript strict mode.
- Keep domain logic out of route components.
- Prefer small, typed service functions over large page-level data logic.
- Keep public reads separate from admin writes.
- Validate external data before writing it to the database.
- Preserve source links and version history for imported or edited records.
- Treat public slugs as permanent identifiers.
- Keep UI responsive, accessible, and usable in dark mode.
